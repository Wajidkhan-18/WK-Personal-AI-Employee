# Plan Generator Skill

Create structured task plans for AI Employee execution.

## When to Use

The Plan Generator is triggered when:
- Complex multi-step task detected in Needs_Action/
- Task requires coordination across multiple systems
- Human approval needed at multiple stages
- Task has dependencies or prerequisites
- Estimated completion time > 10 minutes

## Plan File Structure

```markdown
---
plan_id: PLAN_20260330_143000_001
created: 2026-03-30T14:30:00Z
updated: 2026-03-30T14:30:00Z
status: pending_approval
priority: high
objective: "Generate and send invoice to Client A"
source_file: "EMAIL_invoice_request.md"
estimated_duration: "15 minutes"
requires_approval: true
---

  - To: client_a@email.com
  - Subject: January 2026 Invoice - $1,500

| Step | Depends On | Blocker |
|------|-----------|---------|
| 3 | 1, 2 | None |
| 4 | 3 | None |

| 6 | 5 | None |

## Approval Points

| Step | Action | Threshold | Status |
|------|--------|-----------|--------|
| 5 | Send email | All external emails | ⏳ Pending |

## Completion Criteria

Task is complete when:
1. Invoice PDF generated and saved to /Invoices/
2. Email sent to client_a@email.com
3. Transaction logged in accounting
4. Source email marked as read
5. Plan file moved to /Plans/Done/

If step fails:
- **Step 3 fails:** Use manual invoice creation template
- **Step 5 fails:** Create approval file for manual send
- **Step 6 fails:** Log to /Logs/error_log.md, retry next cycle
```

## Usage

### Create Plan from Action File

```bash
# Auto-generate plan
python scripts/plan_generator.py create --source Needs_Action/EMAIL_001.md

# Create with custom objective
python scripts/plan_generator.py create --source Needs_Action/TASK_001.md \
  --objective "Process client onboarding"

# Create with approval required
python scripts/plan_generator.py create --source Needs_Action/PAYMENT_001.md \
  --requires-approval
```

### Update Plan Progress

```bash
# Update plan status
python scripts/plan_generator.py update --plan Plans/PLAN_001.md \
  --step 3 --status complete

# Mark step as blocked
python scripts/plan_generator.py update --plan Plans/PLAN_001.md \
  --step 4 --status blocked --reason "Awaiting approval"
```

### Generate Plan Report

```bash
# Status of all active plans
python scripts/plan_generator.py status

# Export plan progress
python scripts/plan_generator.py report --plan Plans/PLAN_001.md
```

## Status Values

| Status | Icon | Description |
|--------|------|-------------|
| `pending` | ⏳ | Not started |
| `in_progress` | 🔄 | Currently executing |
| `complete` | ✅ | Successfully finished |
| `blocked` | 🚫 | Cannot proceed (see reason) |
| `failed` | ❌ | Execution failed |

## Error Recovery

### Step Failure Handling

```python
# Pseudocode for step execution
def execute_step(step):
    try:
        result = step.action()
        step.status = "complete"
        return result
    except TransientError:
        step.status = "blocked"
        step.retry_after = now + 5 minutes
        return None
    except FatalError as e:
        step.status = "failed"
        step.error = str(e)
        trigger_rollback(step)
        return None
```

### Rollback Execution

```python
def trigger_rollback(failed_step):
    # Find last known good state
    checkpoint = find_last_checkpoint(failed_step.plan_id)
    
    # Execute rollback steps
    for rollback_action in failed_step.rollback_plan:
        try:
            rollback_action.execute()
        except Exception as e:
            log_error(f"Rollback failed: {e}")
            escalate_to_human()
```

## Troubleshooting

### "Plan stuck in pending"

Check if approval file was created and moved to Approved/.

### "Step execution failed"

Review error in plan file, check MCP server status.

### "Approval expired"

Approvals expire after 24 hours. Create new approval request.
